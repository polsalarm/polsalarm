# Repository Structure

This repo is `polsalarm/polsalarm` — a GitHub **profile repository**. Because it
matches the username, its `README.md` renders on https://github.com/polsalarm.
It doubles as a profile-as-website with a machine-readable (SEO/AEO) layer for
LLMs and answer engines.

```
.
├── README.md                  # Human-facing profile (renders on GitHub profile)
├── CITATION.cff               # How to cite this profile / its projects
├── humans.txt                 # The people behind the profile
├── robots.txt                 # Crawler rules + LLM context pointers
├── sitemap.xml                # URL index for crawlers
├── llms.txt                   # Short machine-readable index (LLMs/agents)
├── llms-full.txt              # Full machine-readable profile
├── llms-ctx-full.txt          # Expanded prose context for answer engines
├── llms-index.json            # Structured JSON index + source hierarchy
├── schema/
│   └── person.jsonld          # Schema.org Person entity graph (JSON-LD)
├── public/                    # Linked documents (web-servable)
│   ├── RECRUITER.md           # One-page recruiter brief
│   ├── FAQ.md                 # Frequently asked questions
│   ├── STACK.md               # Detailed tech stack
│   ├── PROOF.md               # Verifiable awards + metrics
│   ├── AGENTS.md              # Instructions for AI agents
│   ├── LICENSE.md             # License (MIT structure / reserved content)
│   └── llms-full.txt          # Web-served mirror of /llms-full.txt
├── docs/
│   └── STRUCTURE.md           # This file
└── .github/
    ├── POST_MERGE.md          # Post-merge / maintenance checklist
    └── workflows/
        └── snake.yml          # Generates contribution-snake SVG → output branch
```

## Branches

- `main` — source of truth (README + all files above)
- `output` — auto-generated by `snake.yml`; holds the snake SVG consumed by the README

## Update flow

1. Edit content (`README.md`, `public/*`, machine files).
2. Keep `PROOF.md` and `schema/person.jsonld` in sync with any new award/project.
3. Commit + push to `main`. Snake regenerates on push and daily cron.
4. See `.github/POST_MERGE.md` for the post-change checklist.
